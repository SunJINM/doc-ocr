# iFlow 项目规范

## 文档管理规则

- 所有总结文档、方案文档都必须放到 `docs` 目录下
- 文档必须使用中文命名
- 文档格式建议使用 Markdown (.md)

## 项目结构

```
doc-ocr/
├── docs/                 # 文档目录（所有中文文档）
├── src/                  # 源代码
├── tests/                # 测试文件
├── README.md             # 项目说明
└── IFLOW.MD              # 项目规范（本文件）
```

## 文档命名示例

- `试卷解析技术方案.md`
- `OCR模块设计总结.md`
- `项目开发日志.md`
- `API接口文档.md`

## 注意事项

- 禁止在项目根目录创建英文命名的文档文件
- 技术方案、总结、设计文档等必须归类到 docs 目录
- 保持文档结构的清晰和一致性
